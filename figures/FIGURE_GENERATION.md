# Figure Generation Guide

This document describes exactly which script generates each panel of every
manuscript figure, which data files are required, and how to run the code.

All paths are relative to the **repository root** unless otherwise stated.
Data files are not stored in this repository — download them from Zenodo
(see `data/README.md`).

---

## Figure 2 — ABM TME State Space

Figure 2 is assembled from individually generated SVG panels using
`figures/figure_2/generate_figure_2.py`. Each panel must be generated
separately before running the compositor.

Run the compositor from the **repo root**:
```bash
python figures/figure_2/generate_figure_2.py
```

### Panel A — Feature clustermap of ABM state space

**Script:** `code/02_abm_state_space_analysis.marimo.py`
**Run:** `marimo run code/02_abm_state_space_analysis.marimo.py` (or open in marimo)
**Output:** `output/figures/abm/abm_state_space_spatial_statistics_clustermap.svg`

**Required data files:**
| File | Path |
|------|------|
| Clustered ABM windows with state labels | `data/abm/processed/abm_windows_clustered_with_state_label_20260212.csv` |
| Leiden cluster means | `data/abm/processed/abm_leiden_cluster_means.csv` |
| Leiden linkage matrix | `data/abm/processed/abm_full_features_leiden_clusters_linkage_matrix.npy` |
| Leiden distance matrix | `data/abm/processed/abm_full_features_leiden_clusters_distance_matrix.npy` |
| UMAP embedding | `output/objects/abm/abm_full_features_pc_n647_umap_embedding.npy` |

### Panels B–H — UMAP visualizations and simulation trajectories

**Script:** `code/01_abm_generate_embedding.py`
**Run:** `marimo run code/01_abm_generate_embedding.py` (or open in marimo)

> **Note:** `01_abm_generate_embedding.py` is a marimo notebook. The panel
> SVGs are exported via interactive save buttons within the notebook. The
> relevant cells are:
> - Panels B–E: the "Feature over UMAP" cells for `effector_T_cell`,
>   `exhausted_T_cell`, `malignant_epithelial_cell`, and
>   `malignant_epithelial_cell_degree_centrality`
> - Panel F: the "ABM time UMAP" cell
> - Panel G: the "ABM state UMAP" cell
> - Panel H: the "Simulation trajectories" cell (set `num_sims = 3`)

**Outputs:**
| Panel | Output path |
|-------|-------------|
| B | `output/figures/abm/effector_T_cell.svg` |
| C | `output/figures/abm/exhausted_T_cell.svg` |
| D | `output/figures/abm/malignant_epithelial_cell.svg` |
| E | `output/figures/abm/malignant_epithelial_cell_degree_centrality.svg` |
| F | `output/figures/abm/abm_time_umap.svg` |
| G | `output/figures/abm/abm_state_umap.svg` |
| H | `output/figures/abm/simulation_trajectories_over_umap/simulation_trajectories_n3.svg` |

**Required data files:**
| File | Path |
|------|------|
| Simulation parameter trajectories | `data/abm/all_simulations_param_trajectory_20251022.pkl` |

### Panel I — ABM state snapshot montage

**Script:** `code/abm_state_image_examples.marimo.py`
**Run:** `marimo run code/abm_state_image_examples.marimo.py`

This notebook generates interactive HTML visualizations of individual
simulation snapshots for each of the 6 TME states. Each state snapshot
is exported manually (e.g., via browser screenshot or Inkscape).
The six snapshot images are then assembled into a single panel SVG
in Inkscape and saved as:

`output/figures/abm/abm_state_snapshots.svg`

**Required data files:**
| File | Path |
|------|------|
| Simulation parameter trajectories | `data/abm/all_simulations_param_trajectory.pkl` |
| ABM UMAP embedding | `output/objects/abm/abm_full_features_pc_n647_umap_embedding.npy` |
| Simulation snapshot images | `output/figures/abm/state-snapshots/` (generated by notebook) |

## Figure 4 — MIBI Clinical Outcomes

Figure 4 is generated by a single self-contained script.

**Script:** `figures/figure_4/generate_figure_4.py`
**Run from `figures/figure_4/`:**
```bash
cd figures/figure_4
python generate_figure_4.py
```

### Panels A–F

All panels are generated by `generate_figure_4.py` in a single run.
Two additional standalone scripts generate individual panels with extended
statistical output for review:

- `panel_c_tme_recurrence_analysis.py` — Panel C (TME state vs. recurrence)
- `panel_d_e_tme_survival_analysis.py` — Panels D & E (survival analysis)

**Required data files** (all in `figures/figure_4/data/`):
| File | Description |
|------|-------------|
| `clinical_data.csv` | Patient survival and recurrence endpoints (n=19) |
| `mibi_roi_tme_proportions_clinical.csv` | Per-patient TME state proportions with clinical data |
| `mibi_roi_spatial_summaries_state_labeled.csv` | ROI-level spatial network metrics with state labels |
| `mibi_patient_mixing_score_labels.csv` | Per-patient Keren et al. mixing score classification |

## Figure 6 — ABM TME State Space Dynamics and Markov State Model

Figure 6 is generated by a single self-contained script.

**Script:** `figures/figure_6/generate_figure_6.py`
**Run from `figures/figure_6/`:**
```bash
cd figures/figure_6
python generate_figure_6.py
```

> **Note:** This figure was numbered Figure 5 in earlier development versions
> of the code. The figure was renumbered in the final manuscript.
> Data files are named `fig_6*.pkl` (previously `fig_5*.pkl`).

If Panel G data is missing, run `recover_g.py` first to extract it from

```bash
python recover_g.py
python generate_figure_6.py
```
